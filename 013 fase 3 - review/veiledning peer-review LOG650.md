# Veiledning for peer-review-rapport i LOG650

Vurderingskriterier og rapportstruktur — Våren 2026

I LOG650 skal hver gruppe gi en *peer-review* av en annen gruppes prosjektrapport og levere tilbakemeldingen som en kort, skriftlig rapport. Vurderingskriteriene under er en tilpasning av Avdeling for logistikks kriterier for masteroppgaver, og brukes her som rammeverk for *konstruktiv tilbakemelding* – ikke som grunnlag for karaktersetting. Tilbakemeldingen skal være saklig, balansert, konkret, begrunnet og handlingsrettet. AI kan brukes fritt som støtte både i lesefasen og skriveprosessen. Gruppen står ansvarlig for at tilbakemeldingen er faglig forsvarlig og reflekterer gruppens vurderinger.

## 1. Vurderingsområder og kriterier

*Tabell 1: Vurderingsområder og kriterier for peer-review.*

| Område | Kriterier |
|---|---|
| **Innledning** | Bakgrunn og kontekst. |
| | Problemstilling. |
| | Forskningsmål eller forskningsspørsmål. |
| | Studiens betydning og relevans er tydelig forklart, og forskningsmålene framstår som verdt å undersøke. |
| | Sammenheng mellom forskningsmål/-spørsmål og studiens betydning. |
| **Litteraturgjennomgang og teoretisk forankring** | Gjennomgang av relevant forskning innenfor det aktuelle området. |
| | Identifikasjon av teoretiske eller begrepsmessige hull. |
| | Gjennomgang av metoder som er relevante for de oppgitte målene innenfor (eller tilgrensende til) forskningsområdet, og begrunnelse for valgte metoder. |
| **Metode** | Metodevalgene er plausible og sammenhengende. |
| | Detaljer i analytisk rammeverk og framgangsmåte. |
| | Beskrivelse av data brukt i studien (der relevant), inkludert struktur og innsamlingsmetode (simulering, spørreundersøkelse, intervju, observasjon, o.l.). |
| | Validitet, reliabilitet og etiske hensyn er forklart. |
| **Analyse og resultater** | Analysen er korrekt utført. |
| | Resultatene er internt konsistente og logisk knyttet til forskningsspørsmålene og det analytiske rammeverket. |
| **Diskusjon** | Funnene tolkes; både forventede og uventede resultater pekes ut og forklares ved å reflektere over teori, praktisk problembakgrunn, eller tidligere forskning. |
| | Funnene knyttes tilbake til forskningsmålene og forskningsspørsmålene. |
| | Implikasjoner av funnene for praksis, teori eller policy. |
| **Konklusjon** | Konsis oppsummering av sentrale funn. |
| | Studiens bidrag til teori og praksis. |
| | Refleksjon over begrensninger ved studien (rekkevidde, metode, antakelser, o.l.). |
| | Forslag til videre forskning som er forankret i forskningsbakgrunnen og problemstillingen. |
| **Skriveflyt, formelle aspekter og helhetsvurdering** | Klarhet i språk, struktur og organisering. |
| | Korrekt referansebruk (APA 7) og kryssreferanser til figurer og tabeller. |
| | Klarhet og kvalitet på figurer, tabeller og annet visuelt støttemateriale. |
| | Hensiktsmessig bruk av forkortelser, akronymer og nomenklatur. |
| | Originalitet i problemstilling, framgangsmåte eller bidrag. |

Kriterier som åpenbart ikke er relevante for den aktuelle rapporten (f.eks. datainnsamling i en ren litteraturstudie) kan utelates, men dere skal kort begrunne hvorfor.

## 2. Struktur for peer-review-rapporten

Peer-review-rapporten skal være på ca. **2–4 sider**, leveres som **PDF**, og inneholde følgende deler:

1. **Forsideopplysninger.** Gruppe som vurderer, gruppe som blir vurdert, tittel på rapporten som vurderes, dato.

2. **Helhetsinntrykk.** Kort sammendrag (5–10 linjer) av rapportens hovedstyrker og hovedutfordringer.

3. **Områdevis vurdering.** Én underseksjon per vurderingsområde i Tabell 1. For hvert område gir dere en faglig kommentar som – der det er relevant – løfter fram *styrker*, peker på *forbedringspunkter* (med henvisning til konkrete steder i rapporten) og foreslår *konkrete endringer*. Hvis et område er gjennomgående sterkt, holder det å konstatere det; ikke konstruér forbedringsforslag der dere ikke ser reelle behov.
